# Padrões de Comentários no Código

## Boas Práticas

- **Código Auto-Documentado**: Escrever código que se explique através de estrutura e nomenclatura claras
- **Comentários Mínimos e Úteis**: Adicionar comentários concisos e mínimos para explicar seções grandes de lógica de código
- **Não Comentar Mudanças ou Correções**: Não deixar comentários de código que falem sobre mudanças ou correções recentes ou temporárias. Comentários devem ser textos informativos permanentes que sejam relevantes no futuro distante
