# Padrões de Acessibilidade de UI

## Boas Práticas

- **HTML Semântico**: Usar elementos HTML apropriados (nav, main, button, etc.) que transmitam significado para tecnologias assistivas
- **Navegação por Teclado**: Garantir que todos os elementos interativos sejam acessíveis via teclado com indicadores de foco visíveis
- **Contraste de Cor**: Manter proporções de contraste suficientes (4.5:1 para texto normal) e não depender apenas de cor para transmitir informação
- **Texto Alternativo**: Fornecer texto alt descritivo para imagens e labels significativos para todas as entradas de formulário
- **Teste de Screen Reader**: Testar e verificar que todas as views são acessíveis em dispositivos de leitura de tela
- **ARIA Quando Necessário**: Usar atributos ARIA para aprimorar componentes complexos quando HTML semântico não é suficiente
- **Estrutura Lógica de Cabeçalhos**: Usar níveis de cabeçalho (h1-h6) na ordem apropriada para criar um esboço de documento claro
- **Gerenciamento de Foco**: Gerenciar foco apropriadamente em conteúdo dinâmico, modais e aplicações single-page
