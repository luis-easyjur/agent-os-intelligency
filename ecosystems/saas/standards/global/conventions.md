# Convenções Gerais de Desenvolvimento

## Boas Práticas

- **Estrutura de Projeto Consistente**: Organizar arquivos e diretórios em uma estrutura previsível e lógica que os membros da equipe possam navegar facilmente
- **Documentação Clara**: Manter arquivos README atualizados com instruções de setup, visão geral da arquitetura e diretrizes de contribuição
- **Boas Práticas de Controle de Versão**: Usar mensagens de commit claras e descritivas
- **Configuração de Ambiente**: Usar arquivos de configuração separados para diferentes ambientes; nunca commitar secrets ou API keys no controle de versão
- **Respeitar Estrutura Legada**: Não reestruturar código existente sem solicitação explícita; melhorias devem ser incrementais
- **Padrões de Arquivos**: Seguir padrões estabelecidos (`ajax_*.php` para AJAX, `JANELA_*.php` para modais)
